# Bearball Seven 專案規則

## 專案定位與技術

- 本專案是 Bearball Seven 官方網站。
- 網站使用 Astro 開發，以 GitHub 管理原始碼，並由 Cloudflare Pages 部署。
- BearballCastle 是本專案與世界觀中的正式名稱，書寫時應保持此拼法與大小寫。

## 修改與操作規則

- 修改前必須先分析需求，並向使用者說明預計修改哪些檔案。
- 未經使用者明確要求，不得執行 `git push`、merge 或 deploy。
- 完成任何程式或內容修改後，必須執行 `npm run build`，確認專案能成功建置。
- 不要無故新增 npm 套件或導入其他前端框架；只有在需求明確且確有必要時才可考慮。
- 修改範圍應限於完成需求所必要的檔案，避免無關變更。

## 設計原則

- 保持目前乾淨、留白、重視閱讀性的設計風格。
- 新增或調整頁面時，應維持既有網站的視覺語言與閱讀節奏。

## 角色、名稱與世界觀

- Bearball Seven 與肉丸七是同一個角色。
- Xiaotao 與小桃是同一位重要朋友。
- BearballCastle 是正式名稱。
- Creator’s Bible 是世界觀、角色設定與相關敘事的最高依據；若其他內容與其衝突，應以 Creator’s Bible 為準。
